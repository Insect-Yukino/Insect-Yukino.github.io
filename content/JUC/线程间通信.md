我们常见的线程间通信机制大多是 `Object` 提供的 `Object#wait`、`Object#notify` 和 `Object#notifyAll`。

我第一次见到这样的设计是觉得奇怪的，为什么线程之间的协作功能是由 `Object` 类提供的。这时就要提到 Java 的监视器锁。

## Java 的监视器锁

Java 的并发控制是基于 **Monitor** 实现的。而它正是依附于 Object 上。我们时长 `new` 一个对象，但是对象的组成不单单只是我为其设计的属性和方法。

我们也可以使用 `Condition` 提供的 `Condition#await` 和 `Condition#signal` 方法。 

`Thread#sleep` 方法并不会释放线程占用的锁，只会暂停运行指定时间。而 `Object#wait` 会释放当前持有的锁。
