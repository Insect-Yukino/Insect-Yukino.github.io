设计一个成熟的电商系统真难。这里先了解一下电商系统的核心链路，并且给出部分表设计

## 核心链路

### 商品域

涉及商品价格，库存、上下架状态等。

老生常谈的问题有精确定价、精确库存、支持并发访问。

### 交易域

涉及购物车系统、下单操作、锁库存、生成订单等。

要注意防止超卖、订单幂等、高并发下单不崩。

### 支付域

支付单、第三方支付、支付回调。

金额不能错、状态不能乱、重复回调不出问题。

### 履约域

发货、物流、收货确认。

状态可追踪、支持售后。

### 风控 & 对账

防刷单、放重复支付、财务对账。

其中最核心部分，订单系统的表设计如下。

用户表

```sql
CREATE TABLE user (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  username VARCHAR(64) NOT NULL,
  phone VARCHAR(20),
  status TINYINT DEFAULT 1 COMMENT '1-正常 0-禁用',
  created_at DATETIME,
  updated_at DATETIME
);
```

商品 SPU 表（商品大类）
```sql
CREATE TABLE product (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(255) NOT NULL,
  status TINYINT COMMENT '1-上架 0-下架',
  created_at DATETIME,
  updated_at DATETIME
);
```

订单表

```sql
CREATE TABLE `order` (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    order_no VARCHAR(50) UNIQUE,
    user_id BIGINT,
    total_amount DECIMAL(10,2),
    status TINYINT COMMENT '0待支付 1已支付 2已发货 3已完成 4已取消',
    create_time DATETIME,

    INDEX idx_user (user_id)
);
```

SKU 具体的单件商品

```sql
CREATE TABLE sku (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  product_id BIGINT NOT NULL,
  sku_name VARCHAR(255),
  price DECIMAL(10,2) NOT NULL,
  stock INT NOT NULL,
  locked_stock INT DEFAULT 0,
  created_at DATETIME,
  updated_at DATETIME,
  INDEX idx_product_id (product_id)
);
```

购物车
```sql
CREATE TABLE cart (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT,
  sku_id BIGINT,
  quantity INT,
  created_at DATETIME,
  UNIQUE KEY uk_user_sku (user_id, sku_id)
);
```

订单主表表

```sql
CREATE TABLE `order` (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  order_no VARCHAR(64) NOT NULL,
  user_id BIGINT NOT NULL,
  total_amount DECIMAL(10,2) NOT NULL,
  status TINYINT NOT NULL COMMENT '0-待支付 1-已支付 2-已发货 3-已完成 4-已取消',
  created_at DATETIME,
  updated_at DATETIME,
  UNIQUE KEY uk_order_no (order_no),
  INDEX idx_user_id (user_id)
);
```

订单明细

```sql
CREATE TABLE order_item (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  order_id BIGINT NOT NULL,
  sku_id BIGINT NOT NULL,
  price DECIMAL(10,2) NOT NULL,
  quantity INT NOT NULL,
  created_at DATETIME,
  INDEX idx_order_id (order_id)
);
```

库存流水表
```sql
CREATE TABLE stock_log (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  sku_id BIGINT,
  order_no VARCHAR(64),
  change_num INT,
  type TINYINT COMMENT '1-锁库存 2-解锁 3-扣减',
  created_at DATETIME,
  INDEX idx_sku_id (sku_id)
);
```

支付表

```sql
CREATE TABLE payment (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  order_no VARCHAR(64),
  pay_amount DECIMAL(10,2),
  pay_type TINYINT COMMENT '1-支付宝 2-微信',
  status TINYINT COMMENT '0-未支付 1-成功 2-失败',
  trade_no VARCHAR(128),
  created_at DATETIME,
  updated_at DATETIME,
  UNIQUE KEY uk_order_no (order_no)
);
```

下单支付的关键流程

下单流程

```text
1. 校验商品 & 价格
2. 锁库存（stock - n, locked_stock + n）
3. 创建订单（status = 待支付）
4. 返回 order_no
```

支付成功回调
```sql
1. 校验金额 & order_no
2. 修改订单状态为 已支付
3. 扣减 locked_stock
4. 记录库存流水
```

从上面的表设计可以看出，如果要查询用户详细订单订单。需要关联订单表和订单明细表

```sql
SELECT *
FROM `order` o
JOIN order_item oi ON o.id = oi.order_id
WHERE o.user_id = ?
  AND o.order_no = ?;
```

易知，每一个用户关联一个购物车。查用户购物车有什么商品时 sql 如下

```sql
SELECT
  c.sku_id,
  c.quantity,
  s.price,
  s.sku_name,
  p.name AS product_name
FROM cart c
JOIN sku s ON c.sku_id = s.id
JOIN product p ON s.product_id = p.id
WHERE c.user_id = ?;
```
