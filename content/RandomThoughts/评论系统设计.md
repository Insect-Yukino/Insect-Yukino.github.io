一个成熟的评论系统至少要解决：

- 评论发给谁
- 谁发的
- 评论结构
- 状态控制
- 互动
- 性能

评论系统最核心的表设计

评论主表

```sql
CREATE TABLE comment (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  biz_type TINYINT NOT NULL COMMENT '业务类型：1-商品 2-订单 3-文章',
  biz_id BIGINT NOT NULL COMMENT '业务ID，如 product_id',
  user_id BIGINT NOT NULL COMMENT '评论用户',
  parent_id BIGINT DEFAULT 0 COMMENT '父评论ID，0表示一级评论',
  root_id BIGINT DEFAULT 0 COMMENT '根评论ID，0表示一级评论',
  content TEXT NOT NULL,
  status TINYINT DEFAULT 1 COMMENT '1-正常 0-删除 2-审核中',
  like_count INT DEFAULT 0,
  reply_count INT DEFAULT 0,
  created_at DATETIME,
  updated_at DATETIME,
  INDEX idx_biz (biz_type, biz_id),
  INDEX idx_parent (parent_id),
  INDEX idx_user (user_id)
);
```

`biz_type` + `biz_id` 决定了评论发给谁，评论系统可复用。

`parent_id` + `root_id` 支持一级评论、二级回复、楼中楼

评论点赞表

```sql
CREATE TABLE comment_like (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  comment_id BIGINT NOT NULL,
  user_id BIGINT NOT NULL,
  status TINYINT DEFAULT 1 COMMENT '1-点赞 0-取消',
  created_at DATETIME,
  UNIQUE KEY uk_comment_user (comment_id, user_id),
  INDEX idx_comment (comment_id)
);
```

`like_cout` 用于快速统计点赞数。

而该表的作用是为了防止点赞重复、支持取消点赞、支持我是否点赞过。

评论举报表

```sql
CREATE TABLE comment_report (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  comment_id BIGINT NOT NULL,
  user_id BIGINT NOT NULL,
  reason VARCHAR(255),
  status TINYINT DEFAULT 0 COMMENT '0-未处理 1-已处理',
  created_at DATETIME,
  INDEX idx_comment (comment_id)
);
```

查询评论的 sql 

查询商品的一级评论

```sql
SELECT *
FROM comment
WHERE biz_type = 1
  AND biz_id = ?
  AND parent_id = 0
  AND status = 1
ORDER BY created_at DESC
LIMIT 10 OFFSET 0;
```

查询某条评论的回复

```sql
SELECT *
FROM comment
WHERE root_id = ?
  AND parent_id != 0
  AND status = 1
ORDER BY created_at ASC;
```

我们在B站或者抖音中看到的评论效果是非常迅速的。但是如果评论信息直接从 MySQL 中读取，大量的查询请求和写入操作会把 MySQL 干宕机。

因此，这些评论数据主要是从 Redis 中读取的。

在 Redis 中存储每个视频下的所有评论 id

```text
key: comment:list:videoId
value: 评论ID列表（ZSET / LIST）
```

评论内容单独存放

```text
key: comment:list:videoId
value: 评论ID列表（ZSET / LIST）
```

读路径是完全不走数据库
用户发评论时

```text
 服务生成 comment_id
 评论内容写入 Redis
 comment_id 插入 Redis 评论列表
 立即返回成功
```

MySQL 异步存储
```text
Redis → MQ → MySQL
```
