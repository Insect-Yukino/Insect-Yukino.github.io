系统的异常处理是非常重要的，尤其是当我们的系统抛出异常后，有些异常是不能被抛出到用户层的，即不能让用户看到，因为容易被有心之徒窃取资料。

我们今天介绍一下在 Spring 项目中如何处理各个层面的异常。

## Spring MVC 全局异常处理器

首先第一关就是 Spring MVC 的全局异常处理器。

使用起来非常边便捷，只需要使用 `@ControllerAdvice / @RestControllerAdvice` 声明一个全局组件。写法如下：

```java
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(BizException.class)
    public Result<?> handleBiz(BizException e) {
        return Result.fail(e.getCode(), e.getMessage());
    }

    @ExceptionHandler(Exception.class)
    public Result<?> handleOther(Exception e) {
        return Result.fail(500, "系统异常");
    }
}
```

`@ExceptionHandler(Class)` 该注解用于声明处理那中异常。我们一般把项目中的异常分为业务异常和系统异常，业务异常在项目中由程序员手动抛出，配合前端为用户提供友好的提示。

而系统异常就不是程序员控制的了，那是在程序未知情况下抛出的异常，这部分异常通常是由于程序没有设计好造成的。最常见的就是 NPE。

在一个项目中，异常的格式需要我们进行统一，将业务异常派生成一类，系统异常则现需要我们兜底处理。

## Spring Security

如果我们使用的是 Spring Security，那么他异常链中抛出的异常就需要我们单独处理，Spring MVC 的全局异常处理器无法管理这部分异常。

这里主要涉及到两类分异常：

- 认证异常，为登录访问受保护资源，处理该异常的类是 `AuthenticationEntryPoint`
- 授权异常，已登录但权限不足，处理该异常的类是 `AccessDeniedHandler`

### AuthenticationEntryPoint

```java
@Component
public class CustomAuthenticationEntryPoint 
        implements AuthenticationEntryPoint {

    @Override
    public void commence(HttpServletRequest request,
                         HttpServletResponse response,
                         AuthenticationException e)
            throws IOException {

        response.setContentType("application/json;charset=UTF-8");
        response.setStatus(HttpServletResponse.SC_UNAUTHORIZED);

        response.getWriter().write("""
        {
            "code": 401,
            "message": "未登录或登录失效"
        }
        """);
    }
}
```

### AccessDeniedHandler

```java
@Component
public class CustomAccessDeniedHandler 
        implements AccessDeniedHandler {

    @Override
    public void handle(HttpServletRequest request,
                       HttpServletResponse response,
                       AccessDeniedException e)
            throws IOException {

        response.setContentType("application/json;charset=UTF-8");
        response.setStatus(HttpServletResponse.SC_FORBIDDEN);

        response.getWriter().write("""
        {
            "code": 403,
            "message": "权限不足"
        }
        """);
    }
}
```

实现相关处理器之后，在配置中注册

```java
@Bean
public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {

    http
        .exceptionHandling(ex -> ex
            .authenticationEntryPoint(customAuthenticationEntryPoint)
            .accessDeniedHandler(customAccessDeniedHandler)
        );

    return http.build();
}
```

当 Security Filter 链中抛出异常，就会交由这些处理器处理。

在企业中，这些内容可以做如下目录结构

```text
security/
 ├── CustomAuthenticationEntryPoint
 ├── CustomAccessDeniedHandler
 ├── JwtAuthenticationFilter
 └── SecurityConfig
```

## Spring Cloud Gateway 

Spring Cloud Gateway 作为网关层，异常同样需要单独处理。

Spring Cloud Gateway 基于 Spring WebFlux，全局异常处理接口是 `ErrorWebExceptionHandler`。他的默认实现类是 `DefaultErrorWebExceptionHandler`。如果我们要定制网关的异常处理，只需要实现 `ErrorWebExceptionHandler`。

```java
@Component
@Order(-1)
public class GlobalGatewayExceptionHandler 
        implements ErrorWebExceptionHandler {

    private final ObjectMapper objectMapper = new ObjectMapper();

    @Override
    public Mono<Void> handle(ServerWebExchange exchange, Throwable ex) {

        ServerHttpResponse response = exchange.getResponse();
        response.setStatusCode(HttpStatus.OK);
        response.getHeaders().setContentType(MediaType.APPLICATION_JSON);

        Map<String, Object> result = new HashMap<>();
        result.put("code", 500);
        result.put("message", ex.getMessage());

        byte[] bytes;
        try {
            bytes = objectMapper.writeValueAsBytes(result);
        } catch (JsonProcessingException e) {
            bytes = new byte[0];
        }

        DataBuffer buffer = response.bufferFactory().wrap(bytes);
        return response.writeWith(Mono.just(buffer));
    }
}
```

`@Order(-1)` 会使该异常处理器的优先级高于默认处理器。

如果我们自定义网关过滤器

```java
public class JwtFilter implements GlobalFilter {}
```

该过滤器中的异常就会被 `ErrorWebExceptionHandler` 捕获，所以不需要被 `try-catch` 吞掉。
