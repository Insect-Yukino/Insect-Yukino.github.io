在 Java 中，我们经常使用 `String#equals` 比较两个字符串是否相等。但在刷一些力扣题目时，遇到了比较两个字符串大小的问题，头一次见到，所以就查了一下。

字符串大小比较本质上是：**按字典序从左到右逐个字符比较**，比较的依据通常是 **每个字符的编码值**，在 Java 里是 Unicode/UTF-16 的 `char` 值，遇到第一个不同字符就决定大小；如果一直相同，就用 **长度** 决定。

## 1）字典序比较的规则

给两个字符串 `s`、`t`：

1. 从下标 `0` 开始比较：`s[0]` vs `t[0]`
2. 如果相等，继续比较 `s[1]` vs `t[1]` ……
3. **遇到第一个不相等的字符**：
   - 若 `s[i]` 的编码值 < `t[i]` 的编码值，则 `s < t`
   - 反之 `s > t`
4. 如果前 `min(len(s), len(t))` 个字符都相等：
   - 那结果就是 **短的更小**

在 Java 中，有 `String#compareTo` 来比较两个字符串的大小。 

`compareTo` 返回一个整数：

- `= 0`：完全相等
- `< 0`：当前字符串更小
- `> 0`：当前字符串更大
