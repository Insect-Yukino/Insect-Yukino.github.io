今天看 AI 生成代码的时候，对于项目中常量的处理，我敢绝非常优雅，记录一下：

```java
public final class UserProfileKeys {

    private UserProfileKeys() {}

    private static final String DIRTY = "user:profile:dirty:";
    private static final String AGG = "user:profile:agg:";
    private static final String VIEW = "user:profile:view:zset:";
    private static final String SEARCH = "user:profile:search:zset:";
    private static final String VECIOR = "user:profile:vector:";

    public static String getDirtyKey(Long userId) {
        return DIRTY + userId;
    }

    public static String getAggKey(Long userId) {
        return AGG + userId;
    }

    public static String getViewKey(Long userId) {
        return VIEW + userId;
    }

    public static String getSearchKey(Long userId) {
        return SEARCH + userId;
    }

    public static String getVectorKey(Long userId) {
        return VECIOR + userId;
    }
}
```

首先是这个常量类不能被继承，可以声明为 `final`，其次，拼接字符串的时候，可以提供静态方法，这样是不是就不用写那些老长的变量名了。
