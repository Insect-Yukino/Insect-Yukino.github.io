服务监控体系一般分为 4 层：

```text
基础设施层（服务器）
 ↓
运行时 / 容器层
 ↓
应用服务层
 ↓
业务 & 用户层
```

## 第一层：服务本身的监控

**监控对象**：CPU / Load、内存、磁盘 IO、网络流量、主机存活。

**典型的指标有**：CPU 使用率、内存使用率、磁盘剩余空间、TCP 连接数。

主流方案：

- **Prometheus + Node Exporter**

- **Zabbix**

- 云厂商监控（阿里云 / AWS CloudWatch）

## 第二层：运行环境监控

监控对象

- Docker 容器
- JVM
- GC
- 线程数
- 文件句柄

常见工具

- cAdvisor（容器）
- JMX Exporter（Java）
- Micrometer（应用指标）

## 第三层：服务监控

监控内容

| 类型       | 说明     |
| ---------- | -------- |
| QPS        | 请求量   |
| RT         | 响应时间 |
| Error Rate | 错误率   |
| 依赖调用   | 下游服务 |
| 健康状态   | 心跳     |

主流组合：

- **Prometheus**

- **Grafana**

## 第四层：链路追踪 & 日志

链路追踪是为了查看一次请求的调用链路

常用 SkyWalking

日志监控是为了排查为什么出错
常用方案

**ELK Stack**

- Elasticsearch
- Logstash / Filebeat
- Kibana

实现这一套监控的最主流的方案就是 Prometheus 体系

```text
Node Exporter → Prometheus → Grafana
                  ↓
              Alertmanager
```
