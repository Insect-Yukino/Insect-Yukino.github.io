> FTP 服务器是提供文件传输服务的服务器。

FTP = File Transfer Protocol（文件传输协议）

它的作用是：

- 上传文件
- 下载文件
- 浏览服务器目录
- 删除 / 重命名文件

在早期互联网中，如果想：

- 把文件从 A 电脑传到 B 电脑
- 在服务器上放资源供别人下载

就需要一种标准协议。

FTP 就是专门干这个的。

它采用：

```text
客户端 - 服务器模式
```

流程是：

```text
FTP 客户端
     ↓
连接 FTP 服务器
     ↓
登录账号
     ↓
执行文件操作
```

FTP 默认端口

| 类型     | 端口           |
| -------- | -------------- |
| 控制连接 | 21             |
| 数据连接 | 20（主动模式） |

FTP 有两个连接：

- 控制连接（发命令）
- 数据连接（传文件）

FTP 有两种模式

###### 主动模式（Active）

服务器主动连接客户端。

```text
客户端 ← 服务器发起数据连接
```

容易被防火墙拦截。

###### 被动模式（Passive）

客户端主动连接服务器的数据端口。

```text
客户端 → 服务器
```

现在大多数使用被动模式。

FTP 是：

- 明文传输
- 用户名密码明文
- 数据明文

容易被抓包。

FTPS =FTP + SSL/TLS

是加密传输。

SFTP ≠ FTP

它是基于：

SSH 协议

默认端口：

```text
22
```

更安全。

以前常用于：

- 网站上传（把代码传到服务器）
- 下载公共资源
- 企业文件共享

现在很多被：

- HTTP 下载
- 对象存储
- 网盘
- Git

替代。

FTP 它本质是：

> 在服务器上运行一个FTP 服务程序，监听 21 端口，处理文件操作命令。

常见 FTP 服务端软件：

- vsftpd（Linux）
- FileZilla Server
- IIS FTP

简单理解模型

```text
你电脑
  ↓
FTP 协议
  ↓
FTP 服务器
  ↓
服务器磁盘文件系统
```

> FTP 服务器就是一台专门用来远程传文件的服务器程序。
