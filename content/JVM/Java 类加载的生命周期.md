因为你看到的“实例字段有值”，大多数时候不是**类加载阶段**给的，而是 **对象创建阶段**（new 的时候）由 JVM 自动做了“实例字段默认初始化 + 构造器初始化”。

把两件事分开就清楚了：

------

## 1）类加载阶段初始化的是“类变量”（static）

类被初始化（`<clinit>`）时，JVM 会执行：

- `static` 字段的显式赋值
- `static {}` 静态代码块

这一步只针对 **类本身**，跟你 new 多少个对象没直接关系。

------

## 2）new 一个对象时，JVM 一定会给“实例字段”默认值

对象创建大致是：

1. **分配内存**（堆上拿一块内存）
2. **把这块内存清零**（非常关键！）
3. 设置对象头、指向类元数据
4. 执行实例初始化：`<init>`（构造方法 + 实例字段显式赋值 + 实例代码块）

因为有第 2 步“清零”，所以即使你没写任何赋值，实例字段也会有默认值：

- `int/long/short/...` → `0`
- `boolean` → `false`
- `char` → `\u0000`
- 引用类型 → `null`
- `float/double` → `0.0`

这就是你 new 出来的对象“实例字段也会有值”的根本原因：**对象内存先被清零了**。

------

## 3）如果你写了“实例字段赋值/实例代码块”，它发生在构造器之前/期间

比如：

```
class A {
  int x = 10;
  String s = "hi";
  { x = 20; } // 实例代码块
  A() { x = 30; }
}
```

new `A()` 的初始化顺序是：

1. 默认值：`x=0, s=null`（内存清零）
2. 执行实例字段显式赋值 + 实例代码块（按源码顺序）：`x=10, s="hi"`, 然后 `x=20`
3. 执行构造方法：`x=30`

最终：`x=30, s="hi"`

------

## 4）“类加载”和“对象创建”各自负责什么，一句话

- **类加载/类初始化**：只负责 `static` 的东西（类级别一次）
- **new 对象**：负责实例字段默认值 + 执行构造初始化（每个对象一次）
