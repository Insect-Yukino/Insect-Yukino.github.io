当我们按下电源按钮时，主板会先给电源管理芯片一个信号。电源稳定后，各个部件完成供电，随后会向 CPU 发出一个信号：

```text
Power Good
```

它告诉 CPU：现在可以开始执行指令了。

CPU 上电后会做两件事：

1. 把寄存器清零
2. 把指令指针强制指向一个固定地址。

这个地址是硬件写死的，它会映射到一块 ROM/Flash 芯片，里面存放的是：

- BIOS（老式）
- UEFI（现代）

它们的共同点是：不依赖硬盘，也不依赖操作系统，负责把机器带到“可启动”的最早阶段。

BIOS/UEFI 第一阶段会做什么呢？

1. **POST（上电自检）**

- CPU
- 内存
- 显卡
- 键盘

2. 初始化硬件
3. 建立最基础的运行环境

这时，操作系统还完全不存在。

UEFI / BIOS 会按照预设顺序查找可启动设备，典型顺序可能是：`NVMe SSD → SATA SSD → HDD → USB → Network`。

两种经典方式

传统 BIOS（MBR）

- 读取硬盘 **第一个扇区（512 字节）**
- 找到 **Bootloader**
- 跳转执行

UEFI（现代）

- 读取 EFI 分区
- 找到 `.efi` 文件
- 直接加载

常见的 Bootloader：

- GRUB（Linux）
- Windows Boot Manager
- systemd-boot

它的职责只有一个：

> **把操作系统内核加载进内存**

Bootloader 接下来会完成这些事：

- 把内核文件读进内存
- 设置 CPU 到保护模式 / 长模式
- 跳转到内核入口函数

最后跳转到内核入口，例如执行类似 `jmp kernel_entry` 这样的动作。

**这一步之后，固件就彻底退场了**

内核初始化

操作系统内核会依次完成：

1. 初始化内存管理（分页）
2. 初始化中断系统
3. 初始化调度器
4. 识别硬件（驱动）
5. 挂载根文件系统

这一步结束后，内核会做一件非常重要的事：

> **启动第一个用户态进程**

用户空间开始运行

内核启动的第一个进程：

- Linux：`init` / `systemd`
- Windows：`smss.exe`

之后系统会继续拉起用户空间中的关键服务，顺序通常可以概括为：

1. system 服务
2. 登录服务
3. 桌面环境 / GUI

这时你看到的：

- 桌面
- 窗口
- 鼠标
- 浏览器

这些你最终看到的内容，本质上都只是运行在操作系统之上的普通程序。
