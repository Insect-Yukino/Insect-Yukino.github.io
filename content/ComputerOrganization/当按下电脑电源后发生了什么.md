当我们按下电源按钮时，主板会给电源管理芯片一个信号。电压开始输出稳定电压，各个部件供电完成后，向 CPU 发一个信号：

```text
Power Good
```

告诉 CPU 可以开始工作了。

CPU 上电后会做两件事：

1. 把寄存器清零
2. 把指令指针强制指向一个固定地址。

这个地址是硬件写死的，它映射到一块 ROM/Flash 芯片，里面存的是：

- BIOS（老式）
- UEFI（现代）

他们的特点是不依赖硬盘、不依赖操作系统、端点不丢失。

BIOS/UEFI 第一阶段会做什么呢？

1. **POST（上电自检）**

- CPU
- 内存
- 显卡
- 键盘

2. 初始化硬件
3. 建立最基础的运行环境

这时，操作系统还完全不存在。

UEFI / BIOS 会按照顺序查找是否由能启动的系统程序。

```
NVMe SSD → SATA SSD → HDD → USB → Network
```

两种经典方式

传统 BIOS（MBR）

- 读取硬盘 **第一个扇区（512 字节）**
- 找到 **Bootloader**
- 跳转执行

UEFI（现代）

- 读取 EFI 分区
- 找到 `.efi` 文件
- 直接加载

常见的 Bootloader：

- GRUB（Linux）
- Windows Boot Manager
- systemd-boot

它的职责只有一个：

> **把操作系统内核加载进内存**

Bootloader 做完这些事：

- 把内核文件读进内存
- 设置 CPU 到保护模式 / 长模式
- 跳转到内核入口函数

```
jmp kernel_entry
```

**这一步之后，固件就彻底退场了**

内核初始化

操作系统内核会依次完成：

1. 初始化内存管理（分页）
2. 初始化中断系统
3. 初始化调度器
4. 识别硬件（驱动）
5. 挂载根文件系统

这一步结束后，内核会做一件非常重要的事：

> **启动第一个用户态进程**

用户空间开始运行

内核启动的第一个进程：

- Linux：`init` / `systemd`
- Windows：`smss.exe`

之后：

```
system 服务
 ↓
登录服务
 ↓
桌面环境 / GUI
```

这时你看到的：

- 桌面
- 窗口
- 鼠标
- 浏览器

全部是普通程序
