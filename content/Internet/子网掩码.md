前些天面试又被拷打了，是有关网络层面的知识。面试官问了我子网掩码的作用是什么，本人还没有开专业课，回答的支支吾吾的，寄了，现在总结复盘一下。

## 网卡

### 1. 网卡的本质

网卡是设备的网络接口，是设备和网络之间的出入口。

没有网卡，设备就无法通过以太网、Wi-Fi 登方式收发网络数据。

常见的网卡形式有：

- 有线网卡
- 无线网卡
- 服务器上的多口网卡
- 虚拟机的虚拟网卡
- Docker、VPN 创建的虚拟网络接口

### 2. 网卡的作用

网卡主要有这些作用：

- 把设备接入网络
  - 设备通过网卡接入局域网、无线网络或其他网络
- 负责收发数据
  - 应用程序的数据作用要通过网卡发送出去；别人发过来的数据也先到网卡，再交给操作系统。

- 在二层网络里有自己的 MAC 地址
  - 网卡通常带有自己的 MAC 地址，局域网内部的数据帧转发依赖 MAC 地址。

IP 地址会分配在网络接口上，也就是配置在网卡上。

一台设备可以有多个网卡，每个网卡可以有不同的 IP。

## IP 地址

IP 地址是设备在 IP 网络中的逻辑地址，用来标识谁在网络里。

通常不只有一个 IP 地址，它还需要配套

- 子网掩码
- 默认网关
- DNS

因为设备不仅要有地址，还要知道

- 哪些目标属于本地网络
- 哪些目标需要传给网关
- 域名怎么解释

## 子网掩码

### 1. 定义

子网掩码是用来把一个 IP 地址分成两部分的

- 网络部分
- 主机部分

例如 IP：`192.168.1.10`；子网掩码：`255.255.255.0`

这表示前 24 位是网络号。后 8 位是主机号。

也常写成 `192.168.10/24`

这里的 `/24` 和 `255.255.255.0` 是同一个意思。

### 2. 子网掩码的作用

子网掩码最核心的作用是让设备判断目标 IP 是否和自己在同一个子网。

也就是用来判断是直接通信还是交给默认网关转发。

子网掩码会和 IP 地址做按位与运算，用来算出网络地址。

我们现在看到的无论是 IP 还是子网掩码，都是十进制。在运算时需要转换成二进制运算。同时需要注意子网掩码必须是前 n 位都是 1，后 n 位都是 0。

`192.168.1.0/24` 表示的是一个子网/网段

含义是网络地址 `192.168.1.0` 前 24 位是网络号，

一个设备通过**网卡**接入网络，**IP 地址**配置在网卡上，**子网掩码**告诉它哪些地址属于本地子网，哪些地址需要交给**默认网关**，而这个 IP 可以通过**静态配置**手工指定，也可以通过 **DHCP** 动态获取；只有这些参数彼此一致并符合当前网络规划，设备才能稳定、正确地通信。
