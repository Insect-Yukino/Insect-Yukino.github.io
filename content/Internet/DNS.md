今天被问到了 DNS，真是伤心的一天，碰到了不问 Java 技术栈的公司。

## 一、DNS 是什么

DNS 的全称是 Domain Name System。它的作用是把人类容易记住的域名，转换成计算机通信需要的 IP 地址。

我们人类一般容易记住 `www.baidu.com` 这种域名，而计算机真正通信需要用到的是 `110.242.xx.xx` 这种 IP 地址。所以，如果我们使用的是 `www.baidu.com` 的话，就需要 DNS 来做翻译。

## 二、DNS 的层次结构

DNS 不是一台服务器，而是一个分布式、层次化的系统。

比如这个域名：

```http
www.example.com
```

从右往左看层级，

- `.` 根域

- `com` 顶级域
- `example` 二级域
- `www` 主机名 / 子域

### DNS 常见层级

#### 1. 根域

最顶层是根域，用一个点表示：

```bash
.
```

所以完整域名理论上应该是：

```bash
www.example.com.
```

最后那个点就是根域，知识平时省略不写。

根域知道从哪个次级下面查找目标 IP ：

- `.com` 去哪找
- `.cn` 去哪找
- `.org` 去哪找

跟域服务器不直接告诉你某个网站的 IP，它只告诉你下一步该问谁。

#### 2. 顶级域

比如：

- `.com`
- `.cn`
- `.org`
- `.net`
- `.edu`

这一层叫顶级域

#### 3. 二级域

例如 `baidu`

这是通常企业或组织购买和管理的主要域名

#### 4. 子域名 / 主机名

例如 `www.example.com` 中的 `www` 就是子域名或者主机名。

### DNS 查询

DNS 服务器按一下顺序工作：

#### 1. 本地 DNS 解析器 / 递归解析器

#### 2. 根 DNS 服务器

当本地查不到时，会访问根 DNS，它会告诉这个域名属于 `.com`，让你去 `.com` 顶级域服务器找

#### 3. 顶级域 DNS 服务器

它会告诉 `example.com` 这个域名是由哪几个权威服务器负责。

#### 4. 权威 DNS 服务器

这是最关键的一层。

它是真正保存域名记录的服务器，比如：

- `www.example.com -> 93.184.216.34`
- `mail.example.com -> 192.0.2.10`

它会给出最终答案。

 ### 浏览器输入网址后，DNS 是怎么翻译成 IP 的

假设在浏览器输入：

```http
www.example.com
```

浏览器要访问这个网站，必须先知道它的 IP。

整个过程可以分很多步。

### 第 1 步：浏览器先检查缓存

浏览器不会一上来就去问外面的 DNS。它先看自己有没有缓存过这个域名。如果我们刚刚访问过：

- `www.example.com`

浏览器可能还记得它对应的 IP，就直接拿来用。

这叫**浏览器 DNS 缓存**

### 第 2 步：操作系统检查缓存

如果浏览器没有缓存，就会问操作系统。

操作系统也有自己的 DNS 缓存。

例如 Windows、Linux、macOS 都可能缓存最近查过的域名结果。

### 第 3 步：检查 hosts 文件

如果缓存里还没有，系统通常还会看 `hosts` 文件。

`hosts` 文件是本地手工指定 `域名 -> IP` 映射的地方。

例如：

```http
127.0.0.1   localhost
192.168.1.10  test.example.com
```

如果这里写了，就不用再问 DNS 服务器了，直接采用这个结果。

### 第 4 步：把查询发给本地 DNS 服务器

如果本机也不知道，就会向配置好的 DNS 服务器发请求。

这个 DNS 服务器通常是：

- 路由器
- 运营商 DNS
- 公共 DNS
- 企业内部 DNS

它被称为：

- **递归解析器**
- **本地 DNS 服务器**
- **DNS Resolver**

我们的电脑通常只负责提问，真正复杂的查询由它代劳。

### 第 5 步：递归解析器怎么一步步找到 IP

假设本地 DNS 服务器也没有缓存。

它就开始正式查询。

这里一般是**迭代查询**的过程。

#### 第 1 层：问根 DNS 服务器

本地 DNS 服务器先问根服务器：

> “请问 `www.example.com` 的 IP 是多少？”

根服务器不会直接给 IP，它会说：

> “我不知道最终 IP，但我知道 `.com` 的顶级域服务器在哪，你去问它。”

于是返回 `.com` 顶级域服务器的信息。

#### 第 2 层：问 `.com` 顶级域服务器

本地 DNS 服务器再去问 `.com` 服务器：

> “请问 `www.example.com` 的 IP 是多少？”

`.com` 服务器也通常不直接给最终 IP，它会说：

> “`example.com` 这个域名归某几个权威 DNS 管理，你去问它们。”

于是返回 `example.com` 的权威 DNS 服务器地址。

#### 第 3 层：问 `example.com` 的权威 DNS 服务器

本地 DNS 服务器再去问权威服务器：

> “请问 `www.example.com` 的 IP 是多少？”

这次权威服务器会直接回答：

> ```http
> www.example.com = 93.184.216.34
> ```

这就是最终解析结果。

#### 第 4 步：把结果返回给电脑

本地 DNS 服务器拿到结果之后：

1. 先把结果缓存起来
2. 再返回给你的操作系统
3. 操作系统可能再缓存一次
4. 浏览器拿到 IP 后开始连接网站服务器

于是浏览器就能继续：

- 建立 TCP 连接
- 如果是 HTTPS，还会 TLS 握手
- 发送 HTTP 请求
- 获取网页内容

> 面试官还问我根 DNS 有多少个服务器，这里总结一下：
>
> - **根**：只有一个根区（`.`），它是整个 DNS 树的最顶层。
>
> - **顶级**：有很多个顶级域，比如 `.com`、`.net`、`.cn`、`.uk` 等；IANA 持续维护这个列表，而且会随时间变动，不是永远固定的 
>
> 注意这里说的是根区（`.`），**根服务器**通常说的是 **13 个逻辑根服务器名字（A 到 M）**，它的实例也有很多。
