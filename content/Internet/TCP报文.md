今天我怀着悲痛的心情写下这篇博客，记录我面试过程中，计算机网络被拷打的记忆。

面试关的问题是：TCP 报文头中有哪些字段？

我之前只准备过 TCP 的三次握手什么的，对于这方面倒是没有什么了解。只知道在三次握手期间会发送报文，TCP 建立之后，比如像 HTTP 这种应用层协议的内容也会被包裹成 TCP 报文。

经过查阅，TCP 报文主要包含以下几个字段：

1. 源端口（Source Port）：占 2 字节，标识发送方端口。
2. 目的端口（Destination Port）：占 2 字节，标识接收方端口。
3. 序列号（Sequence Number）：占 4 字节，标识数据段中的第一个字节的序号。在连接建立后，序列号帮助接收方按顺序重组数据。
4. 确认号（Acknowledgment Number）：占 4 字节，表示接收方期望接收的下一个字节的序列号，只有当 ACK 标志位为 1 时有效。
5. 数据偏移（Data Offset）：占 4 位，指示 TCP 报文头的长度
6. 保留（Reserved）：占 3 位，保留供将来使用，应该设置为 0.
7. 标志位（Flags）：
   - URG：紧急指针有效，表示有紧急数据。
   - ACK：确认号有效，表示确认数据。
   - PSH：推送标志，表示接收到的数据应该尽快传给应用层。
   - RST：重置连接，表示连接出现异常或需要重建。
   - SYN：同步标志，用于建立连接。
   - FIN：结束标志，表示连接的结束。
8. 窗口大小（Window Size）：占 2 字节，指示接收方的接收窗口大小，用于流量控制。
9. 校验和（Checksum）：占 2 字节，检验 TCP 报文是否在传输过程中出现错误。
10. 紧急指针（Urgent Pointer）：占 2 字节，表示紧急数据的结束位置，只有当 **URG** 标志为 1 时有效。
11. 选项（Options）：可变长度字段，用于携带一些额外的控制信息，如最大报文段长度（MSS）、时间戳、窗口扩大因子等。
12. 填充（Padding）：确保 TCP 报文头的长度是 32 位的倍数。
13. 数据（Data）：TCP 数据部分，应用层数据通过这个字段传输。

## 报文种类

TCP 报文主要根据不同的连接状态或操作目的，可以分为以下几种类型。不同种类的报文头字段也会有所不同：

#### 1. SYN 报文（SYN Packet）

- 用于**建立连接**的请求。
- **SYN** 标志位为 1。
- **确认号**和**数据部分**是空的，通常用于发送初始序列号。
- 例子：在三次握手的第一次，客户端发送的 **SYN** 报文。

字段

- **SYN** 标志位为 1。
- **确认号**：无效。
- **序列号**：随机生成。

#### 2. SYN-ACK 报文（SYN-ACK Packet）

- **服务器回应**客户端的连接请求。
- **SYN** 和 **ACK** 都为 1。
- **确认号**：客户端的序列号 + 1。
- **序列号**：服务器选择的初始序列号。

**字段**

- **SYN** 标志位为 1。
- **ACK** 标志位为 1。
- **序列号**：服务器的初始序列号。
- **确认号**：客户端的序列号 + 1。

#### 3. ACK 报文（ACK Packet）

- 用于**确认连接已建立**，是三次握手的最后一步。
- **ACK** 标志位为 1，表示确认号有效。

字段

- **ACK** 标志位为 1。
- **序列号**：客户端的确认号。
- **确认号**：服务器的序列号 + 1。

#### 4. FIN 报文（FIN Packet）

- 用于**结束连接**，发起方发送 `FIN` 报文，通知对方连接关闭。
- **FIN** 标志位为 1。

**字段**

- **FIN** 标志位为 1。
- **序列号**：需要关闭的连接的最后一个序列号。
- **确认号**：对方确认关闭连接的序列号 + 1。

#### 5. RST 报文（RST Packet）

- 用于**重置连接**，当连接异常时，TCP 会发送 RST 报文来强制关闭连接。
- **RST** 标志位为 1。

字段

- **RST** 标志位为 1。
- **序列号**：不重要，通常是一个随机数。
- **确认号**：无效。

#### 6. PSH 报文（PSH Packet）

- 用于**推送数据**，通知接收方尽快将数据交给应用层，不需要等待缓存数据。
- **PSH** 标志位为 1。

字段

- **PSH** 标志位为 1。
- **序列号** 和 **确认号**：用于标识数据段。

#### 7. 普通数据报文

- 用于**数据传输**，携带应用层数据。数据部分存在于报文中。
- 通常是双向通信中的有效数据传输。

字段

- **ACK** 标志位为 1。
- **序列号** 和 **确认号** 用于数据的顺序和确认。
- **数据**：应用层传输的数据。

>  尽管 TCP 报文头的字段结构相同，但不同类型的报文会根据特定的连接状态和目的设置不同的标志位和确认号、序列号、数据内容等。

对于**承载 HTTP 数据的 TCP 报文**，它们的 **TCP 报文头**依然是遵循 TCP 协议的标准结构。无论是用于 **HTTP 请求**，还是 **HTTP 响应**，这些数据都通过已经建立的 **TCP 连接**传输。**HTTP** 是**应用层协议**，而 **TCP** 是**传输层协议**，HTTP 通过 TCP 进行数据传输。
