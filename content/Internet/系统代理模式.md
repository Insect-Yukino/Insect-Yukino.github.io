**一台没有开启任何代理/VPN 的电脑，流量是怎么走的。**

没开代理时，电脑直接通过默认网关把数据发到互联网。

没有中间软件接管。

假设访问：

```http
https://www.baidu.com
```

整个过程如下：

### 第一步：DNS 解析

浏览器先要知道：

```http
www.baidu.com → 多少 IP？
```

流程：

```text
浏览器
  ↓
操作系统
  ↓
DNS 服务器（一般是路由器/运营商提供）
```

返回：

```http
www.baidu.com → 220.xx.xx.xx
```

### 第二步：建立 TCP 连接

浏览器发起：

```http
目标 IP: 220.xx.xx.xx
端口: 443
```

数据经过电脑内部网络栈

```text
浏览器
  ↓
操作系统 TCP/IP 协议栈
  ↓
网卡驱动
  ↓
物理网卡
```

电脑里有一个：

```text
默认网关（Default Gateway）
```

可以通过命令看到：

```bash
ipconfig
```

会显示：

```bash
默认网关: 192.168.1.1
```

这通常是路由器

### 数据流向

```text
电脑
  ↓
局域网交换机
  ↓
路由器（默认网关）
  ↓
运营商
  ↓
目标服务器
```

没有代理时的路由逻辑是：

操作系统有一张：

```text
路由表
```

可以查看：

```text
route print
```

其中有一条：

```bash
0.0.0.0  → 默认网关
```

意思是：

> 所有不属于本地网络的流量，都发给默认网关。

所以：

没有代理 = 所有流量直接发给路由器。

因为：

- 没有本地软件接管
- 没有改系统代理
- 没有虚拟网卡
- 没有修改路由表

数据只经过：

```text
操作系统网络栈
```

然后物理发出。

离开你家后：

```text
路由器
  ↓
运营商接入网
  ↓
骨干网
  ↓
目标服务器机房
```

中间通过：

- BGP 路由
- 多级路由转发

完全基于 IP 路由协议。

虽然流量是直连，但内容是：

```text
TLS 加密
```

运营商能看到：

- 你访问的 IP
- SNI（域名）
- 数据包大小

但看不到：

- 具体网页内容

为什么有些网站在国内访问慢？

直连情况下：

- DNS 解析可能被污染
- BGP 路由绕路
- 跨境带宽限制

没有代理时，完全受运营商路径控制。

# 系统代理模式

如果开启过 Clash / 代理软件的：

> 「系统代理模式」

它会修改操作系统里的代理设置。

#### Windows 

打开：

```text
设置 → 网络和 Internet → 代理
```

会看到：

- 手动代理设置
- 地址：127.0.0.1
- 端口：7890（常见）

如果这里还开着：说明系统代理仍然启用，所有遵守系统代理的软件都会走代理

还可以命令行：

```bash
netsh winhttp show proxy
```

如果显示：

```bash
127.0.0.1:xxxx
```

说明 WinHTTP 代理仍在。

#### TUN / 虚拟网卡模式

如果开启过 TUN 或 VPN：

会发生：

- 生成一个虚拟网卡
- 修改系统路由表

即使软件退出，有时虚拟网卡还在。

可以查看：

```text
控制面板 → 网络适配器
```

是否有：

- TAP-Windows Adapter
- Clash Tun
- Wintun

查看路由表：

```bash
route print
```

如果默认路由不是你路由器，那就是被代理/虚拟网卡接管。

##### 为什么有时候明明关了代理却上不了网？

常见原因：

1. 系统代理还开着
2. PAC 还在
3. 虚拟网卡还在
4. DNS 被改了
5. 路由没恢复

我自己使用的是 Clash

Clash 是什么？

Clash = 本地代理服务器 + 规则引擎 + 流量转发器

它通常做 3 件事：

#### 1.在本机开启一个本地代理端口

比如：

```http
127.0.0.1:7890
```

这叫：

- HTTP 代理
- SOCKS5 代理

此时：

你的电脑 = 代理客户端
 Clash = 本地代理服务器

#### 2.修改系统代理设置

当你打开系统代理模式时：

Clash 会：

```text
系统设置 → 网络 → 代理
```

自动改成：

```http
HTTP 代理：127.0.0.1:7890
HTTPS 代理：127.0.0.1:7890
```

这意味着，你的浏览器发出的流量会先发给 127.0.0.1，再由 Clash 决定发往哪里

#### 3.根据规则转发流量

Clash 会读取配置文件（YAML）：

例如：

```yaml
rules:
  - DOMAIN-SUFFIX,google.com,Proxy
  - DOMAIN-SUFFIX,baidu.com,DIRECT
```

于是访问 google → 走代理服务器，访问 baidu → 直连

### 真正代理是怎么发生的？

流程是这样的：

```text
浏览器
   ↓
系统代理
   ↓
Clash 本地端口
   ↓
远程代理服务器（比如 VPS）
   ↓
目标网站
```

Clash 只是：

- 把你的流量封装
- 加密
- 转发到远程服务器

真正出国的是远程服务器。

### 它是怎么截获所有流量的？

有几种模式：

#### 模式 1：系统代理模式

只代理：

- 浏览器
- 遵守系统代理的软件

不代理：

- 某些游戏
- 某些客户端

#### 模式 2：TUN 模式（虚拟网卡）

这是高级模式。

Clash 会：

- 创建一个虚拟网卡
- 修改路由表
- 把所有流量导入自己

效果类似 VPN。

此时：

几乎所有流量都会被代理。

#### 模式 3：透明代理（Linux / 路由器）

通过：

- iptables
- 重定向规则

拦截所有 TCP 流量。

### 它怎么能解密 HTTPS？

Clash 默认不能解密 HTTPS，只能转发加密流量

它看到的只是：

```
加密数据包
```

除非开启：

> HTTPS 中间人模式（MITM）

这时：

- 它会生成一个本地根证书
- 你必须安装到系统里
- 才能解密 HTTPS

它和 VPN 的区别

|              | Clash          | VPN      |
| ------------ | -------------- | -------- |
| 工作层级     | 应用层代理     | 网络层   |
| 是否虚拟网卡 | 可选           | 必须     |
| 控制粒度     | 可以按域名规则 | 一般全局 |
