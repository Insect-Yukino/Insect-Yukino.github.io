平时说的网络证书，通常指的是：

> **SSL/TLS 服务器证书**

也就是 HTTPS 用的证书。

当你访问：

```java
https://example.com
```

浏览器会：

1. 服务器发来证书
2. 浏览器验证证书是否合法
3. 验证通过才建立安全连接

# 二、证书和什么有关？

证书本质上和这几样东西有关：

1. 服务器的公钥
2. 域名
3. 证书颁发机构（CA）
4. 浏览器/操作系统的信任库

# 三、浏览器如何验证服务器证书？

假设你访问一个 HTTPS 网站：

### 流程：

1. 服务器发送证书
2. 浏览器检查：
   - 域名是否匹配
   - 是否过期
   - 是否被吊销
   - 是否由受信任的 CA 签发

如果验证失败，就打不开网页或提示安全警告。

# 四、学校机房打不开网页，可能有哪些原因？

#### 1.服务器证书问题

如果网站证书：

- 过期
- 域名不匹配
- 自签名证书

所有浏览器都会报错。

但一般不会只有 IE 打不开。

##### 2.IE 太老，不支持新 TLS

老版本 IE：

- 不支持 TLS 1.2+
- 不支持新的加密算法
- 不支持新的根证书

如果网站启用了现代加密套件 IE 会握手失败，表现为打不开网页。这是 **非常常见的情况**。

##### 电脑根证书过期

浏览器验证证书，必须信任一个根证书。

这些根证书：

- 存在于操作系统里
- 叫做受信任的根证书颁发机构

如果：

- 机房电脑很久没更新
- Windows 没打补丁
- 根证书库过期

就会导致：

> 明明网站证书没问题，但电脑不信任

# 五、电脑证书是怎么来的？

##### 根证书来自操作系统预装

比如 Windows 会内置一批：

- DigiCert
- GlobalSign
- Let's Encrypt 等

##### 系统更新自动更新

Windows Update 会更新根证书库。

如果长期不更新，就可能缺失新 CA。

# 六、为什么 IE 容易出问题？

因为 IE：

- 依赖 Windows 的证书库
- 不像 Chrome 内置证书
- 老版本 TLS 支持差

而且：

IE 已停止维护。

很多新网站默认禁用旧加密算法。
