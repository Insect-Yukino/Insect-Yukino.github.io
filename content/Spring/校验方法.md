## 使用 JSR-303 Bean Validation 字段规则校验

@Validated 不能用在类参数上，所以如果 DTO 需要嵌套校验的话，需要使用 @Valid 注解。

我们如果要使参数上的校验注解生效，不管是 DTO 内部的校验注解。还是直接使用在参数上使用注解。需要在被校验的参数前使用 @Validated 或 @Valid。

@Valid 不能使用在类上，所以如果直接使用 @Validated 标注在类上，那么参数列表中不需要在加上 @Validated @Valid 注解。

首先定义一个校验注解

```java
@Documented
@Retention(RetentionPolicy.RUNTIME)
@Target({ ElementType.PARAMETER, ElementType.FIELD, ElementType.METHOD })
@Constraint(validatedBy = {EnumValidator.class, EnumValueValidator.class})
public @interface EnumValid {
    String message() default "不满足业务条件";

    int[] enumeration() default {};

    Class<?>[] groups() default { };

    Class<? extends Payload>[] payload() default { };
}
```

定义它的校验器

```java
@Slf4j
public class EnumValidator implements ConstraintValidator<EnumValid, BaseEnum> {

    private int[] enums = null;

    @Override
    public void initialize(EnumValid enumValid) {
        this.enums = enumValid.enumeration();
        log.info("payload>>{}",ArrayUtils.toString(enumValid.payload()));
    }

    @Override
    public boolean isValid(BaseEnum em, ConstraintValidatorContext context) {
        // 不做空校验
        if(em == null){
            return true;
        }
        //没有配置枚举值不校验
        if (ArrayUtils.isEmpty(enums)) {
            return true;
        }
        for (int e : enums) {
            if (e == em.getValue()) {
                return true;
            }
        }
        return false;
    }
}
```

```java
@Slf4j
public class EnumValueValidator implements ConstraintValidator<EnumValid, Integer> {

    private int[] enums = null;

    @Override
    public void initialize(EnumValid enumValid) {
        this.enums = enumValid.enumeration();
        log.info("payload>>{}",ArrayUtils.toString(enumValid.payload()));
    }

    @Override
    public boolean isValid(Integer value, ConstraintValidatorContext context) {
        // 不做空校验
        if(value == null){
            return true;
        }
        //没有配置枚举值不校验
        if (ArrayUtils.isEmpty(enums)) {
            return true;
        }
        for (int e : enums) {
            if (e == value) {
                return true;
            }
        }
        return false;
    }
}
```

## 使用自定义接口 \+ AOP 调用 `check()` 方法

这是对**有业务语义的动态校验**

首先定义一个注解

```java
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.METHOD)
public @interface ParamChecker {
}
```

被改注解标注的方法参数会被动态获取

定义一个校验抽象接口

```java
public interface Checker<T> {

    /**
     * 用于实现validation不能校验的数据逻辑
     */
    default void check(){

    }

    default void check(T data){
    }
}
```

定义一个需要被校验的类

```java
@Data
@Schema(description = "章节")
public class CataSaveDTO implements Checker {
    @Schema(description = "章、节、练习id")
    private Long id;

    @Schema(description = "目录类型1：章，2：节，3：测试", example = "1")
    @NotNull(message = "目录类型不能为空")
    private Integer type;

    @Schema(description = "章节练习名称", example = "第一章")
    private String name;

    @Schema(description = "章排序，章一定要传，小节和练习不需要传", example = "1")
    private Integer index;

    @Schema(description = "当前章的小节或练习")
    @Size(min = 1, message = "不能出现空章")
    private List<CataSaveDTO> sections;

    @Override
    public void check() {
        // 名称为空校验
        if (type == CourseConstants.CataType.CHAPTER && StringUtils.isEmpty(name)) {
            throw new BadRequestException(CourseErrorInfo.Msg.COURSE_CATAS_SAVE_NAME_NULL);
        } else if (StringUtils.isEmpty(name)) {
            throw new BadRequestException(CourseErrorInfo.Msg.COURSE_CATAS_SAVE_NAME_NULL2);
        }
        // 名称长度问题
        if (type == CourseConstants.CataType.CHAPTER && name.length() > 30) {
            throw new BadRequestException(CourseErrorInfo.Msg.COURSE_CATAS_SAVE_NAME_SIZE);
        } else if (name.length() > 30) {
            throw new BadRequestException(CourseErrorInfo.Msg.COURSE_CATAS_SAVE_NAME_SIZE2);
        }
        if (CollUtils.isEmpty(sections)) {
            throw new BadRequestException("不能出现空章");
        }
    }
}
```

自定义业务校验逻辑

实现一个 AOP，用以执行校验方法

```java
public class CheckerAspect {

    @Before("@annotation(paramChecker)")
    public void before(JoinPoint joinPoint, ParamChecker paramChecker) {
        Object[] args = joinPoint.getArgs();
        if(ArrayUtils.isNotEmpty(args)){
            //遍历方法参数，参数是否实现了Checker接口
            for (Object arg : args){
                if(arg instanceof Checker) {
                    //调用check方法，校验业务逻辑
                    ((Checker)arg).check();
                }else if(arg instanceof List){
                    //如果参数是一个集合也要校验
                    CollUtils.check((List) arg);
                }
            }
        }
    }
}
```

## 一、核心结论先给出（防混淆版）

### ✅ 1️⃣ `@Valid` 和 `@Validated` 都可以触发参数校验

**前提：必须标在“被校验的参数”或“校验生效的入口”上**

- 标在 **方法参数上**
- 或者 `@Validated` 标在 **Controller 类上**

否则，DTO 里的校验注解 **不会生效**

------

### ✅ 2️⃣ **嵌套校验只能用 `@Valid`**

> 这一点你说得是对的，而且非常关键 ✅

- **DTO 中包含另一个对象 / List**
- 要想让“内部对象”继续校验
   👉 **只能在字段上用 `@Valid`**

❌ `@Validated` **不能用于字段嵌套校验**

------

### ❌ 3️⃣「`@Validated 不能用在类参数上`」这句话不严谨

更准确的说法是：

> `@Validated` **不能用于 DTO 内部字段的嵌套校验**，
>  但**可以用于方法参数和类（Controller / Service）上**

------

### ❌ 4️⃣「`@Valid 不能使用在类上`」——基本正确，但要说清楚

- `@Valid` **不能像 `@Validated` 一样标在 Controller 类上作为统一入口**
- 但它**可以标在方法参数上**

------

## 二、三种常见用法，逐个拆清楚

------

### 场景 1️⃣：**最常见用法（推荐）**

```
@PostMapping("/save")
public void save(@Valid UserDTO dto) {
}
```

✅ 生效
 ✅ DTO 内校验生效
 ✅ **嵌套对象需要 `@Valid`**

```
public class UserDTO {

    @NotBlank
    private String name;

    @Valid              // ⭐ 必须
    private AddressDTO address;
}
```

------

### 场景 2️⃣：**类上统一启用校验（你提到的情况）**

```
@RestController
@Validated
public class UserController {

    @PostMapping("/save")
    public void save(UserDTO dto) {
    }
}
```

✅ 生效
 ✅ 方法参数自动校验
 ✅ 参数上 **不用再写 `@Valid / @Validated`**

> ⚠️ 但：
>  **DTO 内部嵌套对象仍然要用 `@Valid`**

------

### 场景 3️⃣：**直接校验普通参数**

```
@GetMapping("/test")
public void test(
    @NotBlank String name,
    @Min(1) Integer age
) {
}
```

此时：

- 方法参数上必须有：
  - `@Validated`（参数前）
  - 或 Controller 类上有 `@Validated`

否则 ❌ 不生效

------

## 三、`@Valid` vs `@Validated` 一张对照表

| 对比点             | `@Valid`  | `@Validated`     |
| ------------------ | --------- | ---------------- |
| JSR 标准           | ✅         | ❌（Spring 扩展） |
| 用在方法参数       | ✅         | ✅                |
| 用在 Controller 类 | ❌         | ✅                |
| 支持嵌套校验       | ✅（唯一） | ❌                |
| 支持分组校验       | ❌         | ✅                |

------

## 四、把你那段话改成「完全正确版」

> **推荐改写如下：**

> 在 Spring Boot Validation 中，参数校验要生效，必须在校验入口处使用 `@Valid` 或 `@Validated`。
>  `@Validated` 可以标注在 Controller 类或方法参数上，用于统一启用参数校验，并支持分组校验；而 `@Valid` 通常标注在方法参数上。
>  当 DTO 中存在嵌套对象或集合时，必须在对应字段上使用 `@Valid` 才能触发嵌套校验，`@Validated` 无法用于嵌套校验。
>  如果在 Controller 类上已经标注了 `@Validated`，则方法参数上可以不再显式标注 `@Valid` 或 `@Validated`，但 DTO 内部的嵌套字段仍需使用 `@Valid`。
