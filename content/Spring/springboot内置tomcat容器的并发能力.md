Spring Boot 默认使用 Tomcat 的 NIO 模型。它通过一个接收线程池和一个工作线程池来处理并发请求

```markdown
┌────────────────────────────────────┐
│         Tomcat NIO Connector       │
├───────────────┬────────────────────┤
│ Acceptor      │ 负责接收连接（socket.accept） │
│               │ 默认 1~2 个线程               │
├───────────────┼────────────────────┤
│ Worker Threads│ 负责处理每个 HTTP 请求       │
│               │ 默认最大 200 个线程           │
└───────────────┴────────────────────┘
```

每个请求都会被分到一个 worker 线程去执行，因此 Tomcat 的并发能力可以视为 worker 线程数

| 属性                            | Spring Boot 3.x 默认值 | 控制层     | 说明                              |
| ------------------------------- | ---------------------- | ---------- | --------------------------------- |
| `server.tomcat.max-connections` | **8192**               | 网络连接层 | 最大允许连接数（Keep-Alive 也算） |
| `server.tomcat.threads.max`     | 200                    | 线程池层   | 同时能执行请求的最大线程数        |
| `server.tomcat.accept-count`    | 100                    | 队列层     | 排队等待处理的请求数              |

这是 tomcat 的核心配置参数 

tomcat 是上游系统，用于接收网络请求处理网络请求

但是如果下游系统的性能过差，tomcat 在持有工作线程处理的阻塞时间就会很长

导致工作线程迟迟不能够释放，影响整个系统的吞吐量
