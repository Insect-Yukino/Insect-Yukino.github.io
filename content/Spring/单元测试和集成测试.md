单元测试应该简约，不启动 Spring 容器环境，只单纯测试方法是否正确，这样测试十分的快，Spring 启动慢也是出了名的。

可以使用如下代码进行测试

```java
@ExtendWith(MockitoExtension.class)
public class UserServiceTest {

    @InjectMocks
    private UserService userService;

    @Mock
    private UserRepository userRepository;

    @Test
    public void testGetUsers() {
        User user1 = new User();
        user1.setUserId(1);
        user1.setName("雪之下雪乃");
        User user2 = new User();
        user2.setUserId(2);
        user2.setName("牧濑红莉栖");
        List<User> users = Arrays.asList(user1, user2);
        List<User> f = List.of(user1);
        Mockito.when(userRepository.getUsers()).thenReturn(users);
        List<User> res = userService.getUsers();
        MatcherAssert.assertThat(res, Matchers.is(res));
        Mockito.verify(userRepository, Mockito.times(1)).getUsers();
    }
}
```

`@ExtendWith(MockitoExtension.class)` 该注解是 JUnit 的扩展机制，用来把 Mockito 接入 JUnit 5 的测试生命周期

```text
JUnit 5
  ↓
@ExtendWith(MockitoExtension)
  ↓
初始化 Mock / InjectMocks
  ↓
执行 @Test 方法
```

`@Mock` 作用是创建一个 Mockito Mock 对象

Mock 对象：

- **不包含真实逻辑**
- 所有方法默认返回：
  - `null`
  - `0`
  - `false`

我们可以通过以下代码给 Mock 定义行为

```java
when(userRepository.getUsers()).thenReturn(users);
```

或

```java
doThrow(new RuntimeException()).when(userRepository).getUsers();
```

`@InjectMocks` 用于创建被测试对象，并把 Mock 依赖注入进去

注入规则

Mockito 按顺序尝试：

1. **构造方法注入（优先）**
2. setter 注入
3. 字段注入

如果依赖没 mock 会发生什么？

```java
@InjectMocks
private UserService userService;
```

但 `UserService` 依赖 `OrderRepository`

却没有

```java
@Mock 
private OrderRepository orderRepository
```

那这个依赖会是 **null**。调用时直接 `NullPointerException`。

运行时发生

```text
JUnit 5 启动测试
  ↓
MockitoExtension 扫描测试类
  ↓
创建 userRepository 的 Mock
  ↓
创建 userService
  ↓
把 userRepository 注入 userService
  ↓
执行 @Test 方法
```

```text
@ExtendWith(MockitoExtension.class)
@Mock
@InjectMocks
```

这种方式是纯单元测试，不启动 Spring 

如果我们要启动 Spring 环境可以如下

```java
@SpringBootTest
public class UserServiceTest {
    
    @Autowired
    private UserService userService;
    
    @MockBean
    private UserRepository userRepository;

    @Test
    public void testGetUsers() {
        User user1 = new User();
        user1.setUserId(1);
        user1.setName("雪之下雪乃");
        User user2 = new User();
        user2.setUserId(2);
        user2.setName("牧濑红莉栖");
        List<User> users = Arrays.asList(user1, user2);
        List<User> f = List.of(user1);
        Mockito.when(userRepository.getUsers()).thenReturn(users);
        List<User> res = userService.getUsers();
        MatcherAssert.assertThat(res, Matchers.is(res));
        Mockito.verify(userRepository, Mockito.times(1)).getUsers();
    }
}
```

使用 `SpringBootTest` 启动 Spring 环境，注意在 Spring 中要使用 `@MockBean` 注解注入模拟对象。

`@Mock` 和 `@MockBean` 从包名就可以看出来区别，前者是 `org.mockito.Mock;` 后者是 `org.springframework.boot.test.mock.mockito.MockBean;`

使用 `@SpringBootTest` 注解表示加载完整的Spring上下文，如果我们想要单独的测试 Controller 可以使用 `@WebMvcTest(UserController.class)`

在 Repository 层可以使用 `@DataJpaTest`。加载特定层的环境

集成测试可以启动 Spring 环境，启动慢，但可以看整体项目有没有问题。

因为 **`@SpringBootTest` 太重了**：

- 启动慢
- Bean 多
- 测试定位困难

一次只测一层，只加载这一层需要的 Bean

`@SpringBootTest` 是全量集成测试，它会启动完整的 `ApplicationContext`。加载：

加载：

- Controller
- Service
- Repository
- 配置类
- 过滤器、拦截器、AOP

自动配置数据源、事务等

适合测试 Bean 装配是否正确，事务是否生效，多模块协作

而 `@WebMvcTest` 注解是 Controller 层的切片测试

它只会加载 Web 层相关 Bean

- `@Controller`
- `@RestController`
- `ControllerAdvice`
- `@JsonComponent`
- MVC 配置

它不会加载真实的 Service、Repository、数据源。因此可以通过 `@MockBean` 模拟

`@DataJpaTest` 是 JPA Repository 切片测试，如果我们使用 MyBatis 可以使用 `@MybatisTest` 测试，它只会加载：

- Mapper 接口
- MyBatis 配置

默认不加载 Service

适合测试：

- SQL
- resultMap
- 表字段映射

  它默认使用 Spring Boot 中的内嵌数据库，不需要引入外部依赖环境

```java
@MybatisTest
@Sql(
    scripts = {"/schema.sql", "/data.sql"},
    executionPhase = Sql.ExecutionPhase.BEFORE_TEST_METHOD
)
class UserMapperTest {

    @Autowired
    private UserMapper userMapper;

    @Test
    void should_select_user_by_id() {
        User user = userMapper.selectById(1L);

        assertNotNull(user);
        assertEquals(1L, user.getId());
        assertEquals("雪之下雪乃", user.getName());
    }

    @Test
    void should_select_all_users() {
        List<User> users = userMapper.selectAll();

        assertEquals(2, users.size());
        assertEquals("雪之下雪乃", users.get(0).getName());
        assertEquals("牧濑红莉栖", users.get(1).getName());
    }
}
```
