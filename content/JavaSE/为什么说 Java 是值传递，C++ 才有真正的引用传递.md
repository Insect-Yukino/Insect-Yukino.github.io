今天又在社交平台上看到讨论，Java 只有值传递，而 C++ 才有真正的指针传递。之前学过的都忘了，现在趁着这个机会再复习一下。

## Java

Java 确实 **只有值传递**。

它传的这个值分两种：

- 一种是基本类型，传的是数值本身
- 另一种是引用类型，传的是引用的值，也就是对象地址的一个副本

因为 Java 会传引用的副本，因此，我们可以通过这个副本去改对象内部内容，但是 **不能通过这个副本，改变外面那个引用变量本身指向谁**。

## C++

而 C++ 就不同了，它既有值传递，又有 **引用传递**。

其中真正的引用传递是：

```c++
void f(int& x)
```

这里的 `x` 是实参的 **别名**，不是副本。对 `x` 的修改，就是直接修改外面的那个变量。

而指针传递本质上还是传递引用副本，不是引用传递。

最能证明 Java 不是引用传递的例子：

```java
public class Main {
    static class Person {
        String name;
    }

    public static void change(Person p) {
        p = new Person();
        p.name = "Tom";
    }

    public static void main(String[] args) {
        Person p = new Person();
        p.name = "Jack";
        change(p);
        System.out.println(p.name);
    }
}
```

输出还是：

```bash
Jack
```

为什么？

因为方法里这句：

```bash
p = new Person();
```

只是让 **形参 p 这个副本** 指向了新的对象。外面的那个 `p` 还是指向原来的对象。

这正说明 Java 传进去的是引用值的副本，不是外部引用变量本身。如果 Java 是真正的引用传递，那么这里外面的 `p` 也应该被改成指向新对象，但事实没有。

C++ 真正的引用传递是什么样？

```c++
#include <iostream>
using namespace std;

void change(int& x) {
    x = 100;
}

int main() {
    int a = 10;
    change(a);
    cout << a << endl; // 100
}
```

这里的：

```c++
int& x
```

表示 `x` 是 `a` 的别名。

不是复制一个 `a` 的值给 `x`，而是 `x` 和 `a` 本质上绑定到同一块内存

所以：

```c++
x = 100;
```

就是直接改 `a`。

### C++ 底层到底怎么做到引用传递的呢？

以下是搜索结果

#### 1. 语言语义层

从语言语义来说，编译器把引用当成同一对象的另一个名字，而不是指向同一个对象的另一个引用。

这是语言规则决定的。

#### 2. 编译实现层

在编译器实现里，引用通常会被实现成类似隐藏指针的东西，但 **这不是语言层面的定义**，只是常见实现方式。

例如：

```c++
int a = 10;
int& ref = a;
ref = 20;
```

编译器内部可能会把它近似处理成：

```c++
int a = 10;
int* __ref = &a;
*__ref = 20;
```

> **形参相当于是实参的别名，对形参的操作其实就是对实参的操作，在引用传递过程中，被调函数的形式参数虽然也作为局部变量在栈中开辟了内存空间，但是这时存放的是由主调函数放进来的实参变量的地址。被调函数对形参的任何操作都被处理成间接寻址，即通过栈中存放的地址访问主调函数中的实参变量。正因为如此，被调函数对形参做的任何操作都影响了主调函数中的实参变量。**

**引用在语义上不是指针**，但编译器在机器层面，常常可以用地址来实现这种绑定关系。

像我们上面举的 Java 不是引用传递的例子在 C++ 上面就可以实现。
