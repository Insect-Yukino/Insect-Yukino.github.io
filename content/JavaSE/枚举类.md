### 1）枚举的“实例”什么时候创建？

枚举的每个常量（`NEW / PROCESSING / ...`）本质上都是该枚举类的**一个实例对象**。这些实例不是你手动 `new`，而是 JVM 在**枚举类初始化阶段**创建的。

更具体地说，触发枚举类的初始化（`<clinit>`）时，执行顺序大致是：

1. **按源码顺序创建所有枚举常量实例**
    等价于（概念上）：

   ```
   NEW = new OutboxStatusEnum((short)0, "待处理");
   PROCESSING = new OutboxStatusEnum((short)1, "处理中");
   ...
   ```

2. 编译器生成一个数组保存所有枚举常量（常见叫 `$VALUES`）

3. 执行你写的 `static { ... }` 静态代码块（如果有）

4. 类初始化完成

所以：**枚举常量实例是在类初始化时创建的**，比 `static {}` 块更早（按源码顺序）。

> 触发类初始化的时机跟普通类一致：第一次主动使用该类时，比如访问一个枚举常量 `OutboxStatusEnum.NEW`、调用静态方法、反射加载等。

------

### 2）`values()` 是静态方法吗？

是的，**`values()` 是编译器为每个 enum 自动生成的 `public static` 方法**。

它的作用是返回该枚举的所有常量数组（通常返回的是一个副本/clone，避免你改内部数组）。

你平时写：

```
for (OutboxStatusEnum e : OutboxStatusEnum.values()) { ... }
```

这就是在调用一个静态方法。

同时还会生成另一个静态方法：

- `public static OutboxStatusEnum valueOf(String name)`
   这个是 `Enum` 基类提供机制、编译器也会生成桥接，按名字取枚举常量。

------

### 3）一个容易记的总结

- **枚举常量 = 静态 final 实例**，在类初始化时创建
- **`values()` = 编译器生成的静态方法**，返回所有枚举常量
- 所以在 `static {}` 里调用 `values()` 没问题，因为常量实例已经创建完了
