## 构造方法是什么

**构造方法（Constructor）** 是一种**在对象创建时自动执行的特殊方法**，主要用于：

- 初始化对象的实例字段
- 建立对象的初始不变式
- 保证对象在可用之前处于合法状态

> 构造方法不是用来创建对象的，而是用来**初始化对象**的。

### C++ 中的构造方法

#### C++ 构造方法的调用时机

在 C++ 中：

- 构造方法在**对象被创建时**自动调用
- 对象的创建方式决定了：
  - 内存分配位置（栈 / 堆）
  - 构造与析构的生命周期

#### C++ 构造方法的作用

**C++ 构造方法负责初始化对象，而不是分配内存。**

内存分配发生在构造方法之前：

- **内存由编译器或运行时分配**
- 构造方法只是在已分配的内存上执行初始化逻辑

C++ 对象在栈/堆上的区别
```cpp
Class c;        // 栈上分配
Class* p = new Class(); // 堆上分配
```

- `Class c;`
  - 对象本身在 **栈上**
  - 构造函数被调用
  - 生命周期随作用域结束

- `new Class()`
  - 对象在 **堆上**
  - 构造函数被调用
  - 需要手动 `delete`

### Java 中的构造方法

#### Java 构造方法的调用时机

在 Java 中：

- 构造方法在执行 `new` 时自动调用
- 每次 `new`：
  - 都会创建一个新对象
  - 都会调用某个构造方法

> Java 中，对象内存由 JVM 在堆上分配；构造方法是在这块已分配的内存上执行初始化逻辑。

#### Java 对象的内存分配时机是什么时候

Java 对象的内存分配发生在 `new` 指令执行时，且一定发生在任何构造方法执行之前。

当我们执行下面的代码时：

```java
MyObject obj = new MyObject();
```

这一行代码在 JVM 中是这样的：

##### 1. 执行 `new` 指令，分配内存

- JVM 在 **堆（Heap）** 中：
  - 找到一块**足够大的连续内存**
  - 用来存放 `MyObject` 的所有实例字段
- 这一步 **就是对象内存分配的时机**

**此时，对象“已经存在于堆中”了，但还不可用。**

##### 2. 内存清零

- JVM 会将这块内存 **清零**
- 所有实例字段变为默认值：
  - `int → 0`
  - `boolean → false`
  - `引用 → null`

这是 Java 字段有默认值的根本原因。

##### 3. 设置对象头

JVM 会在对象内存中写入：

- 对象所属的 **类元数据指针**
- 锁信息
- GC 年龄等

##### 4. 执行构造方法链

到这里执行构造方法，开始初始化类

```java
MyObject() {
    // 构造方法体
}
```

##### 5. 返回对象引用

- 构造方法执行完成
- JVM 将堆中对象的引用赋值给变量 `obj`

```
obj → 指向堆中的对象
```

## 子类的构造方法

无论是 **C++ 还是 Java**：

> **子类在构造时，必须先完成父类的构造，再构造子类自身部分。**

虽然调用了两次构造方法，但只会产生一个对象。这是因为类的内存布局是这样的：

```text
Child 对象（只有一个）
┌──────────────────────────┐
│ Parent 的实例字段        │  ← 父类部分
│ Parent 的初始化逻辑      │
├──────────────────────────┤
│ Child 的实例字段         │  ← 子类部分
│ Child 的初始化逻辑       │
└──────────────────────────┘
```

JVM 做的事情是：

1. **一次性分配一块内存**（足够放下 Parent + Child 的所有实例字段）
2. 将这块内存标记为 **Child 类型的对象**
3. 按顺序调用构造方法：

- 先调用 `Parent()` → 初始化父类部分
- 再调用 `Child()` → 初始化子类部分

### 那为什么一定要先调用父类构造？

原因很简单：

> **子类对象包含父类对象的状态，父类状态必须先初始化**

否则会出现：

- 子类使用了父类字段
- 但父类字段还没初始化

所以 Java 语言规范明确规定：

> **子类构造方法第一行必须是 `super(...)`（显式或隐式）；或编译器自动插入 `super()`**

C++ 需要在初始化列表中调用父类方法
```cpp
class Child {
    public:
    	Child() : Parent() {
            
        }
}
```
