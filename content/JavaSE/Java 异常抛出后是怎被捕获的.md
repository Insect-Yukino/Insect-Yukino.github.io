当 Java 程序抛出异常后，这个异常会不断向上抛出，直到被 `catch` 捕获。

我们经常使用的 Spring 对向上抛出的异常会进行包装，比如我们常见的数据库异常就被包装了。

我在 Spring 项目里遇到异常后，经常看到控制台最上面打印的异常是 Spring 框架的异常。而我能看懂的异常一般在最下面。

Java 异常的打印顺序是由异常链控制的，最终由 **`Throwable#printStackTrace()` 的打印算法** 决定控制台上的异常打印顺序。
