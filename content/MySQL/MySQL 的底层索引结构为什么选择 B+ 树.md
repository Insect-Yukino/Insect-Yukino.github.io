MySQL 的数据保存在磁盘上。磁盘 IO 的开销是十分巨大的，因此，磁盘访问的次数就决定了数据库的性能。而 B+ 树仅仅三层节点就可以为2000万+ 数据建立索引。当访问数据的时候，3 次 IO 可以查询大量数据。因此 MySQL 选择了 B+ 树。

相比于 Java 中的 HashMap，HashMap 的数据是保存在内存中的，内存访问不存在 IO 问题。

**为什么 HashMap 不选择 B+ 树**

Redis 使用跳表等一些数据结构来存储数据。为什么不使用 B+ 树。
