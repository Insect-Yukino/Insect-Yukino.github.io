索引是为了提高查询效率而使用的一种数据结构。在数据库中，数据是存储在磁盘上的，因此 IO 操作是最影响查询效率的，一次 IO 操作就需要等待大量时间，差内存操作好几个两级。因此，数据库的索引就要使用可以减少 IO 操作的结构。这时 B+ 树就成了首选，它凭借着超低的 IO 次数战胜了红黑树、hash 等一众数据结构。

我们最常见的 MySQL 的就是使用 B+ 树数据结构

而 Oracle 等其他数据库使用的索引结构是 B+ 树的变种。因为索引结构的差异，像对于 MySQL 的一些性质，比如 ID 自增的话数据库的效率更高，索引失效等一些内容对于其他数据库来说反而是无效的。

为什么不用跳表，反而是使用 B+ 树呢

和 B 树相比，B+ 树的插入删除效率更高，并且 B 树由于非叶子节点会存储行数据，B 树会更胖，读取的 IO 次数更高。B+ 树有链表，更有利于范围查询。

MySQL 的规则是，会将字符串转换为数字。因此在遇到字符串索引和数字构造条件时，MySQL 默认会调用函数将字符串索引编程数字，

因为索引是按照原始数据构建的，当对索引进行操作后在条件查询，操作后的索引没有索引，因此不会走索引查询

深度分页问题，MySQL 再读取需要的十条数据前，需要读取之前的所有数据，并且再扔掉。
