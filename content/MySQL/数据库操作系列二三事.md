1. 在企业级项目中，一般禁止使用自增主键，会影响数据迁移等操作。推荐使用 UUID v7
2. 不同数据库之间的类型转换条件是不同的。我们熟知的 MySQL 默认会隐式转换，它在 `WHERE varvhar = int` 不会报错，而 PostgreSQL 系列却会直接抛出错误，不允许发生隐式转换。因此我们可以使用 `column::text` 或者 `CAST(val AS VARCHAR)` 等操作解决
3. 使用逻辑删除时，做好设置一个业务无关的主键，这样在新增数据的时候，不会和之前逻辑删除的数据冲突。
4. 在使用 MyBatis-Plus 的过程中，需要注意它的条件构造器，如果不使用 `condition`，遇到空值时 MyBatis-Plus 会拼接 `WHERE column = null` 这会导致什么值都查不到
