```java
class Solution {
    public int coinChange(int[] coins, int amount) {
        // 动态规划，记录每一步金额能达到的个数 +1 是为了表示当金额为 0 时
        int max = amount + 1;
        int[] dp = new int[amount + 1];
        Arrays.fill(dp, max);
        dp[0] = 0; 
        for (int i = 1; i <= amount; i++) {
            // 遍历硬币
            for (int j = 0; j < coins.length; j++) {
                // 如果硬币的大小小于当前值
                if (coins[j] <= i) {
                    dp[i] = Math.min(dp[i], dp[i - coins[j]] + 1);
                }
            }
        }
        return dp[amount] > amount ? -1 : dp[amount];
    }
}
```

零钱兑换这个题是典型的动态规划。

其最重要的状态转移方程

```java
for (int j = 0; j < coins.length; j++) {
    // 如果硬币的大小小于当前值
    if (coins[j] <= i) {
        dp[i] = Math.min(dp[i], dp[i - coins[j]] + 1);
    }
}
```

我们要清楚

> 在零钱兑换问题中，并不是所有金额都会被动态规划“构造”出来，只有那些能由硬币面值直接或间接组合得到的金额才是可达状态，其余状态在 DP 转移中自然会被忽略。

## 二、dp 中并不是每个下标都会“生效”

虽然我们定义了：

```
dp[i] = 凑出金额 i 的最少硬币数
```

但实际上，dp[i] 分成两类：

### ✅ 可达状态

- 能由 coins 中的硬币组合得到
- dp[i] 会被更新为一个 `< max` 的值

### ❌ 不可达状态

- 无法由 coins 组合得到
- dp[i] 会一直保持初始化值 `max`

📌 **DP 并不是强行“凑”每个 i，而是在“筛选哪些 i 是能被凑出来的”**

------

## 三、为什么“和硬币金额等额的数值”最重要？

### 1. 最基础的可达状态就是硬币本身

假设：

```
coins = [2, 5]
```

那么最先能确定的可达状态是：

```
dp[2] = 1
dp[5] = 1
```

因为：

```
dp[2] = dp[2 - 2] + 1 = dp[0] + 1
dp[5] = dp[5 - 5] + 1 = dp[0] + 1
```

👉 **这些是“起点状态”**

------

### 2. 后续所有可达金额，都是从这些起点“扩散”出来的

比如：

```
dp[7] = dp[7 - 5] + 1 = dp[2] + 1
dp[9] = dp[9 - 2] + 1 = dp[7] + 1
```

所以你可以把 DP 想成：

> **从硬币面值开始，在金额轴上不断向右“铺路”**

------

## 四、不可达状态为什么不会“误伤”结果？

关键在这一句：

```
dp[i] = Math.min(dp[i], dp[i - coin] + 1);
```

如果 `i - coin` 是不可达的：

```
dp[i - coin] = max
dp[i - coin] + 1 > max
```

于是：

```
dp[i] = Math.min(dp[i], 一个更大的数)
```

👉 dp[i] **不会被更新**

📌 不可达状态就像“断路”，不会参与扩展

------

## 五、用一个你这句话特别贴合的例子

```
coins = [4, 6]
amount = 7
```

### 可达状态分析：

- dp[4] = 1
- dp[6] = 1

其他：

- dp[1], dp[2], dp[3], dp[5], dp[7] → 全不可达

DP 表会变成：

```
i:   0 1 2 3 4 5 6 7
dp:  0 x x x 1 x 1 x
```

👉 最终：

```
dp[7] > amount → return -1
```

**完全符合你的直觉判断**

------

## 六、为什么 dp[0] 是“唯一的根”？

```
dp[0] = 0;
```

它的意义是：

> “金额为 0 是已知可达的起点”

然后：

- dp[coin] = dp[0] + 1
- dp[coin + coin] = dp[coin] + 1
- …

📌 **dp[0] → 硬币面值 → 更大金额**

这是一棵“金额生成树”。
