**KMP** 算法用来解决在一个主串中高效的查找一个模式串第一次出现的位置，类似实现 Java 中的 `indexOf()` 的功能

它的核心思想是当匹配失败时利用已经匹配成功的部分信息，避免从头重新匹配

例如：
主串 `S = "abcabcabcd"` 模式串 `P = "abcabcd"`
当匹配到 `S[5] != P[5]` 时，暴力算法会回退主串指针到 `S[1]`，而 KMP 利用 **部分匹配表（next 数组）**，可以直接跳到合适的位置。

`next[i]` 表示：

> 模式串中，**以 i 结尾的子串** 的最长前后缀相等的长度。

例如模式串 `"ababca"`：

| i    | 子串(前 i 个字符) | 最长相等前后缀长度 | next[i] |
| ---- | ----------------- | ------------------ | ------- |
| 0    | a                 | 0                  | 0       |
| 1    | ab                | 0                  | 0       |
| 2    | aba               | 1 ("a")            | 1       |
| 3    | abab              | 2 ("ab")           | 2       |
| 4    | ababc             | 0                  | 0       |
| 5    | ababca            | 1 ("a")            | 1       |

因此 `next = [0, 0, 1, 2, 0, 1]`

前缀是什么，后缀是什么

最长相等前后缀是什么

是由主串的后缀匹配模板串的前缀

对模式串 `P`：

> `next[i]` 表示 ——
>  `P[0..i]` 这个子串的 **最长相等前后缀长度**。

也就是说：

- 如果 `next[i] = k`
   代表 `P[0..k-1] == P[i-k+1..i]`
- 当匹配到 `P[i]` 时出错，可以把模式串**往右滑动 (i - k + 1) 位**，
   因为前缀部分 `P[0..k-1]` 已经等于主串中最后的后缀部分。

最大匹配前缀后缀永远是以模板字符第一个字母开头的，因为失配前面的字符串都是匹配的

假设主串 `S` 和模式串 `P` 匹配到了某个位置：

```
S: a b a b a b c
P: a b a b c
            ↑
           这里失配
```

此时：

- 前面 `j` 个字符匹配成功：`P[0..j-1] == S[i-j+1..i]`
- 第 `j` 个字符 `P[j]` 与 `S[i]` 不匹配。

问题是：

> 模式串应该滑到哪儿？
>  也就是说：`j` 应该回退到哪里继续比？

------

## 🧠 二、暴力匹配的浪费在哪里？

暴力算法每次失配后，直接把 `P` 从头开始匹配（`j = 0`）。
 但其实，我们知道前面这 `j` 个字符中，
 有一部分前缀和后缀是**相等的**。

比如 `"abab"`：

```
前缀: a, ab, aba
后缀: b, ab, bab
最长相等前后缀 = "ab"
```

这说明：

> `P[0..1] ("ab")` 和 `P[2..3] ("ab")` 是一样的。

所以，当前面 `"abab"` 已经匹配完之后失配，
 我们没必要从头匹配，
 因为 `"ab"` 这部分信息可以复用！

------

## 🔍 三、为什么最大相等前后缀就是回退位置

看这个关键关系：

```
P[0..j-1]    ← 已匹配成功的部分
前缀 = P[0..k-1]
后缀 = P[j-k..j-1]
next[j-1] = k
```

因为前缀和后缀相等，所以：

> 我们只要让 P 的前缀 “P[0..k-1]”
>  对齐到主串中刚刚匹配的后缀 “S[i-k+1..i]”，
>  下一次比较就能接着往下匹配！

这正是：

```
j = next[j - 1];
```

它表示：

> “让模式串从第 k 个字符开始匹配（跳过前 k 个），
>  因为这 k 个前缀在主串中已经出现过了。”

------

## ⚙️ 四、为什么必须是“从开头第一个字符开始的前缀”

你提到的这句话非常关键👇

> “是因为最大匹配前后缀是从开头第一个字符一个一个加上去的，而不是不带第一个字母的前缀。”

✅ 完全正确！

我们在构造 `next` 数组时，是这么做的：

```
if (p.charAt(i) == p.charAt(j)) j++;
next[i] = j;
```

这意味着：

- `j` 永远表示的是“**以 P[0] 开头**的前缀长度”；
- 它不是从中间起的部分（比如从 `P[1]` 开头的不算前缀）。

这点很重要，因为：

> 我们只相信“真正从模式串第一个字符开始的前缀”，
>  才能与主串中已经匹配的那段后缀 **完全对齐**。

如果允许“跳过第一个字符”的伪前缀（例如 `"bab"`），
 那它在主串中根本没有出现过对应的匹配区域，
 对齐就会错乱。

------

## 🧮 五、直观图例（为什么 j = next[j - 1] 有效）

假设模式串：

```
P = a b a b c
```

next 数组为：

```
[0, 0, 1, 2, 0]
```

当我们匹配到：

```
S: a b a b a b c
P: a b a b c
            ↑ 失配
```

此时 `j = 4`，`next[3] = 2`。

我们让 `j = 2`，相当于：

```
S: a b a b a b c
        a b a b c
```

这里对齐的 `"ab"` 恰好是：

- 模式串前缀 `"ab"`；
- 主串后缀 `"ab"`；
- 两者完全相等 ✅

所以匹配可以从这里继续，不需要主串回退。
