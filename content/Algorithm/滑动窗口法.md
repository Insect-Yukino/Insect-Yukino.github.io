自己的题解：

```java
class Solution {
    public int lengthOfLongestSubstring(String s) {
        // 用一个 Map 记录出现过的字符和它的索引位置
        Map<Character, Integer> map = new HashMap<>();
        // 记录左边界的位置和窗口最大长度
        int left = 0, max = 0;
        // 将字符串变成字符数组
        char[] arr = s.toCharArray();

        // 移动右边界的位置
        for (int right = 0; right < arr.length; right++) {
            char c = arr[right];
            // 判断该字符是否存在当下窗口中
            if (map.containsKey(c)) {
                // 更新左边界的位置
                left = Math.max(left, map.get(c) + 1); // 防止 left 回退
            }
            map.put(c, right); // 每次都要更新位置
            // 更新滑动窗口最大长度
            max = Math.max(max, right - left + 1);
        }

        return max;
    }
}
```

> 上述代码中，我们可以发现，我们并没有清理 `map` 数组中的元素，但是滑动窗口的长度却是改变的，这样我们在 `map#containsKey()` 的时候不会判断到已经被抛弃的元素吗。这个问题就需要看 `left = Math#max(left, map#get(c) + 1)` 的部分。它只会更新最大的 left，并不会更新被丢弃的。防止 `left` 回退。

力扣官方：

```java
class Solution {
    public int lengthOfLongestSubstring(String s) {
        // 哈希集合，记录每个字符是否出现过。这个就代表一个滑动窗口
        Set<Character> occ = new HashSet<Character>();
        int n = s.length();
        // 右指针，初始值为 -1，相当于我们在字符串的左边界的左侧，还没有开始移动
        int rk = -1, ans = 0;
        // 外层循环用来遍历字符数组
        for (int i = 0; i < n; ++i) {
            // 下一次遍历的时候会丢弃上一次中保存的字符
            if (i != 0) {
                // 左指针向右移动一格，移除一个字符
                occ.remove(s.charAt(i - 1));
            }
            // 不断遍历右指针，判断在当前窗口是否出现过相同的字符
            while (rk + 1 < n && !occ.contains(s.charAt(rk + 1))) {
                // 不断地移动右指针
                occ.add(s.charAt(rk + 1));
                ++rk;
            }
            // 第 i 到 rk 个字符是一个极长的无重复字符子串
            ans = Math.max(ans, rk - i + 1);
        }
        return ans;
    }
}
```

> 力扣官方的解法是将 `Set` 作为一个窗口，在遍历的过程中，不断更新这个窗口的大小和容量。
