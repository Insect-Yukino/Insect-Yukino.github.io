# 文章写作规范

这份文档只说明一件事：

**为了让博客页面稳定、规范地显示，Markdown 文章应该怎么写。**

本文档不涉及首页维护、分类顺序、背景图配置等站点维护内容。  
这些内容请看 [README.md](d:/Project/blog/README.md)。

## 1. 基本原则

1. 一篇文章只讲一个主题，标题尽量明确。
2. 标题层级不要跳级。
3. 代码块一定要写正确的语言类型。
4. 表格只放适合表格表达的信息，不要塞超长段落。
5. 图片统一放到 `static/images/` 下，再在 Markdown 中引用。
6. 引用、列表、代码块前后要留空行。

## 2. 一篇文章的推荐结构

推荐结构如下：

```md
+++
date = '2026-06-01T20:00:00+08:00'
draft = false
title = '文章标题'
+++

文章开场简介。

## 第一部分

正文段落。

### 子主题

正文段落。

```json
{
  "demo": true
}
```

## 第二部分

正文段落。
```

## 3. Front Matter 规范

每篇文章文件开头统一写：

```toml
+++
date = '2026-06-01T20:00:00+08:00'
draft = false
title = '文章标题'
+++
```

要求：

1. `title` 必填。
2. `date` 必填。
3. 发布文章时 `draft = false`。

## 4. 标题规范

### 推荐层级

推荐只使用这几层：

```md
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
```

### 不建议的写法

1. 不建议正文里使用多个 `# 一级标题`
   页面主标题已经由 front matter 的 `title` 负责。

2. 不建议标题跳级

不推荐：

```md
## 第二层
#### 第四层
```

推荐：

```md
## 第二层
### 第三层
#### 第四层
```

3. 不建议把整句话都写成很长的标题

不推荐：

```md
### 如果我们在这里要讨论的是 ES 中复杂聚合查询与 pipeline 的组合使用方式
```

推荐：

```md
### Pipeline 聚合
```

然后在正文解释细节。

### 标题使用建议

1. `##` 用来分大章节
2. `###` 用来分子主题
3. `####` 用来分知识点或小节
4. `#####` 尽量少用，只在确实需要更细层级时使用

## 5. 段落规范

### 普通段落

段落之间空一行：

```md
第一段内容。

第二段内容。
```

### 强调

行内强调统一使用：

```md
**重点**
`关键字`
```

不建议整段大面积加粗。

## 6. 列表规范

### 无序列表

统一使用 `-`：

```md
- 第一项
- 第二项
- 第三项
```

### 有序列表

统一使用数字：

```md
1. 第一步
2. 第二步
3. 第三步
```

### 列表中的代码

推荐：

```md
- `sum` 求和
- `avg` 平均值
- `max` 最大值
```

## 7. 代码块规范

这是最重要的一部分。

### 代码块必须使用 fenced code block

统一使用三反引号：

```md
```json
{
  "name": "demo"
}
```
```

### 代码块前后必须空一行

推荐：

```md
下面是示例：

```json
{
  "demo": true
}
```

接下来继续解释。
```

### 代码块语言类型必须写正确

常用类型如下：

```text
json
http
java
bash
yaml
xml
markdown
text
sql
js
```

### 常见场景应该怎么写

#### JSON

推荐：

```md
```json
{
  "query": {
    "match": {
      "title": "手机"
    }
  }
}
```
```

#### HTTP 请求示例

如果你要表达的是“请求行 + 请求体”，推荐仍然使用 `http`：

```md
```http
POST /index/_search
{
  "query": {
    "match_all": {}
  }
}
```
```

但要注意：

1. `http` 更适合展示 REST 请求示例
2. 如果整段内容本质上只是 JSON，不要为了 ES 场景强行写成 `http`

例如下面这种就更适合写成 `json`：

```md
```json
{
  "aggs": {
    "by_status": {
      "terms": {
        "field": "status"
      }
    }
  }
}
```
```

#### Shell / 命令行

```md
```bash
hugo server
```
```

#### Java

```md
```java
public class Demo {
    public static void main(String[] args) {
        System.out.println("Hello");
    }
}
```
```

#### 纯文本流程

像这种结构图、执行顺序图：

```text
query
 -> bucket
   -> metric
     -> pipeline
```

推荐写成：

```md
```text
query
 -> bucket
   -> metric
     -> pipeline
```
```

不要为了“看起来像代码”随便写成 `tex`、`markdown`、`http`。

### 不要拼错语言类型

错误示例：

```md
```josn
```
```

正确示例：

```md
```json
```
```

### 不确定类型时，用 `text`

如果你不知道应该写什么类型，宁可写：

```md
```text
你的内容
```
```

也不要随便乱写一种语言。

## 8. 表格规范

### 推荐写法

```md
| 功能         | 底层结构          |
| ------------ | ----------------- |
| match / term | 倒排索引          |
| sort / agg   | DocValues         |
| `_source`    | 压缩 JSON         |
| **knn**      | **HNSW 向量索引** |
```

### 表格适合放什么

适合：

1. 对比项
2. 字段说明
3. 参数表
4. 简短映射关系

### 表格不适合放什么

不适合：

1. 超长段落
2. 多层嵌套列表
3. 大段代码

这些内容建议改成普通列表或分节说明。

## 9. 引用规范

统一使用：

```md
> 这是引用内容
```

引用块前后保留空行。

如果引用内包含多段内容，也要正常分段。

## 10. 图片规范

### 图片存放位置

推荐放到：

```text
static/images/posts/<分类名>/
```

例如：

```text
static/images/posts/elasticsearch/query-flow.png
```

### Markdown 引用方式

```md
![查询流程图](/images/posts/elasticsearch/query-flow.png)
```

### 图片使用建议

1. 文件名用英文或拼音，避免空格
2. 尽量按分类建目录
3. 图片 alt 不要留空

## 11. 引用代码、关键字、字段名的规范

### 行内代码

字段名、方法名、命令名统一使用反引号：

```md
`terms`
`bucket_selector`
`_source`
`hugo server`
```

### 强调词

重点概念使用加粗：

```md
**倒排索引**
**主分片**
```

## 12. 写作时最容易出问题的地方

### 1. 标题层级乱跳

错误：

```md
## 大标题
#### 小标题
```

### 2. 代码块语言标记错误

错误：

```md
```josn
```
```

### 3. 明明是 JSON，却写成别的类型

错误：

```md
```http
{
  "query": {}
}
```
```

如果没有请求行，推荐直接写成 `json`。

### 4. 代码块前后不空行

错误：

```md
这是一段解释
```json
{
  "a": 1
}
```
下一段
```

### 5. 表格里放太长内容

会导致显示不整齐，也会让移动端体验变差。

## 13. 推荐写作模板

你以后写新文章时，推荐直接按下面这个格式：

```md
+++
date = '2026-06-01T20:00:00+08:00'
draft = false
title = '文章标题'
+++

文章开场说明，用 1 到 3 段说明背景和主题。

## 一、核心概念

先解释概念。

### 1. 基础定义

正文说明。

```text
简单流程或结构
```

### 2. 示例

```json
{
  "demo": true
}
```

## 二、实际过程

正文说明。

### 1. 请求示例

```http
POST /index/_search
{
  "query": {
    "match_all": {}
  }
}
```

### 2. 返回结果

```json
{
  "hits": {
    "total": 10
  }
}
```

## 三、总结

最后用几条列表总结：

- 结论 1
- 结论 2
- 结论 3
```

## 14. 一句话版规则

以后写文章时，只记住下面几条就够了：

1. 正文只用 `##` 到 `####` 为主，别乱跳级。
2. 代码块前后空一行。
3. 代码块语言类型一定写对，拿不准就写 `text`。
4. JSON 就写 `json`，不要乱写。
5. 表格只放简短信息。
6. 图片统一放 `static/images/posts/...`。
